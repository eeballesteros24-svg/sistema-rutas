🚀 Guía de Despliegue

Sistema de Rutas — Cómo publicar y distribuir la app a todo el personal

📦 Parte 1 — Publicar en GitHub Pages

GitHub Pages es gratuito y es exactamente lo que necesitas: convierte tu repositorio en un sitio web al que cualquier persona puede acceder con un link.

1 Sube los archivos al repositorio

Tu repositorio debe tener estos archivos en la raíz (carpeta principal):

📁 tu-repositorio/
├── index.html      ← archivo principal
├── manifest.json
├── sw.js
├── icon-192.png    ← necesitas este ícono
└── icon-512.png    ← y este
⚠️ Los íconos icon-192.png e icon-512.png son necesarios para instalar la app en el celular. Puedes usar cualquier imagen PNG renombrada, o crear un ícono gratuito en favicon.io
2 Activar GitHub Pages

En tu repositorio de GitHub:

  • Haz clic en Settings (engranaje arriba a la derecha)
  • En el menú izquierdo busca Pages
  • En Source, selecciona Deploy from a branch
  • En Branch, selecciona main y carpeta / (root)
  • Haz clic en Save
ℹ️ Espera 1-2 minutos. GitHub te mostrará la URL de tu app, algo como:

https://tu-usuario.github.io/nombre-del-repo/
3 Verificar que funciona

Abre la URL en tu celular. Deberías ver la app funcionando. Si ves un error 404, espera unos minutos más y recarga.

✅ Cada vez que actualices index.html y hagas commit/push, GitHub Pages lo publicará automáticamente en menos de 2 minutos.

📱 Parte 2 — Instalar como app en el celular

La app está configurada como PWA (Progressive Web App), lo que significa que el personal puede instalarla en su celular sin necesidad de Play Store ni App Store. Se comporta exactamente como una app nativa.

En Android (Chrome)

1 Abrir el link en Chrome

El usuario abre el link de GitHub Pages en Chrome. Después de unos segundos, Chrome muestra automáticamente un banner en la parte inferior que dice "Agregar a pantalla de inicio".

Si no aparece el banner, tocan los 3 puntos ⋮ arriba a la derecha → "Agregar a pantalla de inicio".

✅ La app aparece en la pantalla de inicio como cualquier otra app, con el ícono y nombre "Sistema de Rutas". Se abre sin la barra del navegador.

En iPhone / iPad (Safari)

1 Abrir en Safari (obligatorio)

En iOS la instalación solo funciona desde Safari, no Chrome. El usuario debe:

  • Abrir el link en Safari
  • Tocar el ícono de compartir (cuadrado con flecha hacia arriba)
  • Bajar y tocar "Agregar a pantalla de inicio"
  • Confirmar con Agregar
ℹ️ En iOS la app funciona perfectamente pero no puede recibir notificaciones push. Para este caso de uso (solicitar rutas) no es necesario.

📢 Parte 3 — Cómo distribuir al personal

Opción A — Compartir el link por WhatsApp

La forma más simple. Copia el link de GitHub Pages y envíalo al grupo de WhatsApp del personal con estas instrucciones:

Hola equipo 👋

Para solicitar tu ruta de transporte, usa esta app:
👉 https://tu-usuario.github.io/nombre-del-repo/

Para instalarla en tu celular:
📱 Android: abre el link en Chrome → toca los 3 puntos → "Agregar a pantalla de inicio"
🍎 iPhone: abre el link en Safari → botón compartir → "Agregar a pantalla de inicio"

Tu cédula es tu identificación para ingresar.
Opción B — Generar un código QR (recomendado)

Un código QR es ideal para imprimir y pegar en carteleras, neveras o el área de descanso. El personal solo lo escanea y entra directo a la app.

Cómo generarlo gratis:

  • Entra a qr-code-generator.com o qrcode-monkey.com
  • Pega tu URL de GitHub Pages
  • Descarga el QR en PNG o PDF
  • Imprímelo y pégalo donde el personal pueda escanearlo

⚠️ Consideración importante sobre los datos

Los datos se guardan en el dispositivo (localStorage). Esto significa que si el admin exporta el Excel desde su celular y otro admin lo hace desde otro celular, cada uno ve solo los datos registrados en ese dispositivo.

Para este sistema funcione bien con múltiples personas: define un solo dispositivo como "servidor" (ej: el computador del administrador o un celular dedicado) y que el personal use ese dispositivo para solicitar, o bien que todas las solicitudes se vean solo en Admin al exportar el Excel desde el dispositivo del admin.

Si en el futuro necesitan sincronización real entre dispositivos (que todos vean lo mismo en tiempo real), el siguiente paso sería conectar a Firebase (base de datos gratuita en la nube). Es una mejora futura sencilla de implementar.
